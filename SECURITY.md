# Security Policy

## Reporting A Vulnerability

Do not open a public issue for security-sensitive bugs.

Use GitHub private vulnerability reporting for this repository if it is enabled. If it is not enabled yet, open a private security advisory draft or contact the maintainer through a private channel before sharing details publicly.

When you report an issue, include:

- the affected package name and version
- the impact
- clear reproduction steps
- any schema, configuration, or sample code needed to verify the issue

## Scope

This repository publishes official extras packages for Bight.ts. Core runtime issues belong in `bight-core`. CLI or docs issues belong in `bight-extras` unless the root cause is clearly inside one of the packages here.
